# Security Policy

## Reporting a vulnerability

Please do not open public GitHub issues for security-sensitive defects.

Until a dedicated security contact is added, open a private report through GitHub security advisories if enabled for the repository, or contact the maintainer directly.

## Scope

Security reports are especially useful for:

- unsafe shell argument handling
- path traversal or arbitrary file overwrite
- malicious file execution during conversion
- insecure bundling of third-party binaries
